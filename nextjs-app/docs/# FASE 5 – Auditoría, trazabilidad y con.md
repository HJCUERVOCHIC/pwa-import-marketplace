# FASE 5 – Auditoría, trazabilidad y control de acceso

## Objetivo
Garantizar trazabilidad completa del sistema.

## Alcance
- Historial de cambios.
- Control de acceso por rol.

## Requerimientos funcionales
- Historial de cambios de:
  - pedidos (estado, periodo_venta si aplica)
  - ítems
  - pagos
- Registro de usuario y fecha.
- Roles:
  - admin
  - operaciones
  - finanzas
  - lectura

## Entregables
- Tablas de auditoría.
- Políticas de permisos.

## Criterios de aceptación
- Cada cambio financiero queda auditado.
